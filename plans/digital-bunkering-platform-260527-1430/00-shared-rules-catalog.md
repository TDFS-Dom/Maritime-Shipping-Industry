# 00 — Shared Rules Catalog

> Tất cả rules/patterns dùng chung giữa các modules. Modules chỉ REFERENCE tới đây bằng Rule ID, KHÔNG copy-paste.

---

## Platform Rules (PLAT-NNN)

| ID | Rule | Mô tả | Định nghĩa chi tiết |
|---|---|---|---|
| PLAT-001 | API Base Path | Tất cả endpoints prefix `/api/v1/` | platform-core §6 |
| PLAT-002 | Timestamp Format | ISO 8601 `yyyy-MM-dd'T'HH:mm:ss.SSSXXX`, store TIMESTAMPTZ | platform-core §6.5 |
| PLAT-003 | ID Format | UUIDv7 (time-ordered), generated by application | platform-core §6.6 |
| PLAT-004 | Soft Delete | `deleted_at` column, queries append `WHERE deleted_at IS NULL` | platform-core §6.4 |
| PLAT-005 | Workspace Scoping | All queries filtered by `workspace_id` from JWT claims + RLS | platform-core §5 |
| PLAT-006 | Optimistic Locking | `version INTEGER NOT NULL DEFAULT 0`, increment on update, 409 on conflict | platform-core §6.7 |

## Authentication Rules (AUTHN-NNN)

| ID | Rule | Mô tả |
|---|---|---|
| AUTHN-001 | JWT Bearer | `Authorization: Bearer {token}`, RS256, 15 min expiry |
| AUTHN-002 | Workspace Context | Every authenticated request scoped to workspace from JWT claim |
| AUTHN-003 | Role-Based Access | RBAC with permissions `{module}.{action}` |
| AUTHN-004 | Rate Limiting | 60 req/min per user, 200 req/min per workspace |

## CRUD Rules (CRUD-NNN)

| ID | Rule | Mô tả |
|---|---|---|
| CRUD-001 | List Defaults | Offset pagination: `page` (0-based) + `size` (default 20, max 100). Response: `PaginatedResponse<T>` |
| CRUD-002 | Get Defaults | GET /{id}. 404 if not found OR not in workspace |
| CRUD-003 | Create Defaults | POST, response 201, Bean validation on input |
| CRUD-004 | Update Defaults | PATCH /{id}, partial update, optimistic lock check |
| CRUD-005 | Delete Defaults | DELETE /{id}, soft delete (PLAT-004), response 204 |

## Error Rules (ERR-NNN)

| ID | Rule | Format |
|---|---|---|
| ERR-001 | Standard Error Format | `{ "code": "ERROR_CODE", "message": "...", "errors": [...] }` |
| ERR-002 | HTTP Status Mapping | 400=validation, 401=unauth, 403=forbidden, 404=not found, 409=conflict, 422=business rule, 429=rate limit, 500=internal |
| ERR-003 | No Information Leak | 404 for both "not found" and "not in your workspace" |

## Validation Rules (VAL-NNN)

| ID | Rule | Mô tả |
|---|---|---|
| VAL-001 | IMO Number | Regex `^[0-9]{7}$`, exactly 7 digits |
| VAL-002 | Fuel Code | Must exist in fuel-grades registry and be ACTIVE |
| VAL-003 | Quantity MT | `DECIMAL(10,3)`, > 0, display as 3 decimal places |
| VAL-004 | Delivery Window | `start >= now + 24h`, `end > start`, `end - start >= 4h` |
| VAL-005 | String Defaults | `name`: max 255, `description/notes`: max 2000, `reason`: min 10 max 1000 |

## Domain Rules (DOM-NNN)

| ID | Rule | Mô tả |
|---|---|---|
| DOM-001 | MFM Authoritative | MFM reading is authoritative measurement — no manual override (SS 648) |
| DOM-002 | Dual-party Signing | eBDN requires barge crew sign FIRST, then vessel CE (SS 648) |
| DOM-003 | Delivery Method | SS 648/600 operations: delivery method MUST = SHIP_TO_SHIP |
| DOM-004 | Fuel Code Consistency | `nomination.fuel_code = delivery.fuel_code = ebdn.fuel_code` (SS 709) |
| DOM-005 | B2G Submission | Submit eBDN to MPA within required timeframe after signing |
| DOM-006 | Sanctions Pre-check | Sanctions screening MUST complete before nomination can be confirmed |
| DOM-007 | Sample Retention | MARPOL samples retained minimum 12 months (SS 524) |
| DOM-008 | eBDN Immutability | eBDN is immutable after FULLY_SIGNED — no field changes allowed |

## Notification Rules (NOTIF-NNN)

| ID | Rule | Mô tả |
|---|---|---|
| NOTIF-001 | Async Dispatch | Notifications sent via message queue, not synchronous |
| NOTIF-002 | Channels | IN_APP (default), EMAIL, PUSH, SMS (URGENT only) |
| NOTIF-003 | State Change Notify | All business state changes trigger notification to relevant parties |

## Audit Rules (AUDIT-NNN)

| ID | Rule | Mô tả |
|---|---|---|
| AUDIT-001 | All CUD Logged | Every create/update/delete logged in audit_logs |
| AUDIT-002 | Immutable Logs | Audit logs cannot be edited or deleted |
| AUDIT-003 | 7-year Retention | Minimum retention per MPA regulatory requirement |
| AUDIT-004 | State Change Detail | Log old_values + new_values for every state transition |

## Cách sử dụng

```markdown
**Shared Rules:** PLAT-005 (Workspace scoping), CRUD-001 (List defaults), ERR-002 (HTTP codes)

**Chỉ ghi KHÁC BIỆT so với shared rules:**
- Sort: `delivery_date ASC` thay vì default `created_at DESC`
- Max page size: 50 (thay vì default 100)
```
